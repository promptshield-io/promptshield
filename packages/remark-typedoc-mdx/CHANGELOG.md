# remark-typedoc-mdx

## 0.1.0

### Minor Changes

- [`183ff70`](https://github.com/react18-tools/turbo-forge/commit/183ff70a4fb46d18f7da4e66a3eab1d3e1a76d69) Thanks [@mayank1513](https://github.com/mayank1513)! - ### Added

  - **Hierarchical Table Rendering**: Introduced a sophisticated nesting logic for the `parametersAsTable` option.
  - **Context-Aware Processing**: The plugin now differentiates between top-level function arguments, nested object properties, and method signatures within return objects.
  - **Relational Mapping**:
  - **Functions**: Arguments (H3) containing nested properties (H4) are now automatically grouped into a properties table under the argument header.
  - **Complex Returns**: Methods found within a `Returns` section now receive their own sub-context, including dedicated "Parameters" and "Returns" tables for that specific method.

  - **Smart Content Extraction**: Improved heuristic to separate "Type" definitions from "Description" paragraphs, ensuring clean, three-column tables (Name, Type, Description).

  ### Changed

  - **Parameters UI**: Deprecated the flat "dot-notation" naming (e.g., `config.level`) in favor of nested property tables to reduce visual noise and horizontal scrolling.
  - **Signature Normalization**: Enhanced `normalizeSignatures` to better handle blockquoted function signatures typically generated by TypeDoc.
  - **TypeScript Definitions**: Updated internal MDAST node handling to be more robust when traversing deeply nested `RootContent`.

  ### Fixed

  - **Ghost Rows**: Fixed an issue where structural headers (like "Parameters") would generate empty table rows when followed by deeper sub-headers.
  - **Method Parameter Confusion**: Resolved a bug where method parameters were incorrectly flattened into the parent object's namespace (e.g., changing `warn.m` back to a scoped `m` parameter within the `warn()` section).

  ### Documentation

  - Updated README to showcase the new nested table structure.
  - Added examples for complex `createLogger`-style function signatures.

  ***

  ### Internal Note for the Team

  We decided to keep this within the main `remark-typedoc-mdx` library rather than fragmenting into a new package. This allows us to leverage the existing `moduleType` detection to apply specific table strategies (Interface vs. Function) seamlessly.
